# UI and platform architecture

This document describes **cross-cutting presentation and navigation mechanisms** in the Red-Pill Relational Suite so you can reuse or rebuild them in another app. Assessment-specific scoring lives in each `*-engine.js` and `*-data*` module; this file focuses on **shell behavior**: themes, motion, gestures, question controls, modals, suite chrome, and store builds.

---

## 1. Tech model

- **Vanilla** HTML, CSS, and ES modules (no framework, no bundler required for authoring).
- **Capacitor 8** wraps the same `www/` assets for Android; native plugins are loaded via vendored ESM under `shared/vendor/` (generated by `scripts/copy-to-www.js` from `node_modules`).
- **State** is mostly `localStorage` (progress, results, theme keys, entitlement flags). There is no backend in the default product.

---

## 2. Global styling and themes

### Base stylesheet

- **`style.css`**: Universal layout, components, cosmic gradient shell, assessment layouts, suite gates, paywall block (`.android-premium-paywall`), locked nav states, etc.
- **`fonts/fonts.css`**: Local **Inter** (no Google Fonts dependency for offline / Play).

### Theme overlay system

- **Mechanism:** Each page includes `<link id="style-overlay" rel="stylesheet" href="" title="Theme overlay">` after `style.css`. Optional overlays live in **`style/{id}.css`** (e.g. `light`, `forge`, `neomorphism`).
- **Script:** **`shared/style-switcher.js`** (non-module IIFE):
  - Reads/writes **`localStorage`** key `redpill-style-overlay`.
  - Sets `link#style-overlay` `href` to `./style/<id>.css` or clears it for default.
  - **`?style=<id>`** query param applies once, persists to `localStorage`, then is stripped from the URL.
  - Maps **legacy** theme ids (e.g. old marketing names) to the current three overlays via `LEGACY_THEME_MAP`.
- **UI:** A `<select>` is injected only when **`#style-switcher-container`** exists (footer on pages that include it, e.g. `index.html`). Theme choice still applies on pages **without** the selector because overlay + storage run on every page.
- **Font scaling:** Second `<select>` writes **`redpill-font-scale`** to `localStorage` and sets CSS variable **`--font-scale`** on `document.documentElement` (default `1`). All typography should respect this variable if you extend the system.

### Background

- **`shared/background.js`**: WebGL **nebula** on `#nebula-canvas`. Disabled when `prefers-reduced-motion: reduce`, missing WebGL, or coarse-pointer / mobile heuristic (CSS gradient remains).
- **`shared/style-switcher.js`** and **`shared/background.js`** are plain `<script src>` tags (not `type="module"`) so they run everywhere without import maps.

## 3. Swipe navigation

- **Module:** **`shared/swipe-nav.js`** (ES module, auto-inits on load).
- **Behavior:** Horizontal swipe changes page in a **fixed cycle**:  
  `index.html` → `archetype.html` → `temperament.html` → `attraction.html` → `relationship.html` → (wrap).  
  When the suite is complete with a consistent gender, **`integrated-map.html`** is inserted into the cycle after `relationship.html` (see `getSwipeCyclePages()` + `getSuiteCompletion()`).
- **Thresholds:** `SWIPE_COMMIT_PX` (56), direction lock vs scroll (`DIRECTION_LOCK_PX`, `VERTICAL_DOMINANCE`).
- **Reduced motion:** If `prefers-reduced-motion: reduce`, uses **instant** navigation without body transform animation.
- **Guards:** Ignores swipes starting on `input`, `textarea`, `select`, `button`, `[contenteditable]`; ignores horizontally scrollable ancestors (carousels, wide tables).
- **Integration:** Included as `<script type="module" src="./shared/swipe-nav.js"></script>` on main suite pages.

## 4. Sliders, scales, and value allocation

The engines support multiple **question types**; two important patterns are **numeric scales** and **`value_allocation`**.

### Legacy 1–10 (and similar)

- Many items use a **single numeric** answer (often 1–10) stored as a number in the engine’s `answers` / `responses` map.

### `value_allocation` (multi-slider / split 100%)

- **Payload:** `{ type: 'value_allocation', allocationPercents: number[], allocationTotal?: 100 }` per item. Buckets usually sum to 100.
- **Data:** Declared in assessment data (`attraction-data.js`, temperament dimensions, archetype questions, relationship items where converted). See **`docs/MULTI_SLIDER_REFACTOR_SPEC.md`** for schema and projection rules into legacy scalars.
- **Rendering:** Each engine’s question renderer builds paired **range inputs** or equivalent UI so users adjust percentages; normalization ensures totals. Scoring maps allocations to a **scalar** where needed via bucket anchors (see spec).
- **Scoring:** Engines use helpers (e.g. temperament `getTemperamentPolarityNormalizedAnswer`) to interpret either a number or a `value_allocation` object consistently.

When porting: keep the **canonical payload** in sync with export/report code in **`shared/export-utils.js`** (reports serialize answers for HTML download).

## 5. Engine UI state machine

- **`shared/engine-ui-controller.js`**: Small class that toggles visibility using **`hidden`** (and optional `active` class) on configured selectors for states like **`idle`**, **`assessment`**, **`results`**.
- Each `*-engine.js` constructs an `EngineUIController` with a map of which sections to show/hide (intro, action buttons, questionnaire, results).

---

## 6. Modals and alerts

- **`shared/confirm-modal.js`**: **Promise-based** `showConfirm` / `showAlert` with in-DOM styled boxes (avoids native `alert`/`confirm` branding on hosted domains). Used for gates, errors, and post-restore messaging.

---

## 7. Suite progression, nav locks, and home progress

- **`shared/suite-completion.js`**: Derives **stage gates** (Archetype complete → Polarity unlocked → Attraction unlocked), gender alignment, and messages. Relationship is **not** in the sequential chain.
- **`shared/suite-nav-gates.js`**: Adds **`data-suite-locked`**, tooltips, and visual lock on **header / bottom nav / home grid** links to Polarity and Attraction when prerequisites fail; on **native Android**, also when the **Play entitlement** for Polarity+Attraction is missing. Listens for **`redpill-premium-changed`** to refresh.
- **`shared/suite-index.js`**: Renders **suite progress list** on the home page (`#suite-progress-list`) and shows **integrated map** CTA when all three core assessments complete with matching gender.
- **Inline gates:** `temperament.html` / `attraction.html` include `#suiteStageGateInline` blocks; engines dim `#actionButtonsWrap` with `.suite-action-locked` when blocked.

## 8. Android monetization (Polarity + Attraction)

- **`shared/premium-entitlement.js`**: `isNativeAndroid()`, Play Billing via **`@capgo/native-purchases`**, `localStorage` cache, paywall UI hooks, **`restorePurchases`**, and events.
- **Paywall markup:** `#androidPremiumPaywall` in Polarity/Attraction `actionButtonsSection` (see `style.css`).
- **Detail:** **`docs/ANDROID_IAP.md`**.

Web builds never hit the paywall (`Capacitor.getPlatform() !== 'android'`).

---

## 9. Build and asset pipeline

- **`scripts/copy-to-www.js`**: Deletes and repopulates **`www/`** from a whitelist of root files; regenerates **`shared/vendor/`** (Capacitor core + billing plugin ESM with rewritten imports) for unbundled Android WebView loads.
- **`npm run copy:www`**: Refresh `www/` + vendor.
- **`npm run cap:sync`**: `copy:www` then `npx cap sync android`.

---

## 10. Performance and diagnostics

- **`shared/performance-monitor.js`**: Optional instrumentation (where included).
- **`shared/debug-reporter.js`**: Engine-attachable logging hooks.

---

## 11. Porting checklist (new app)

1. Keep **`style.css`** + **`fonts/`** + optional **`style/`** overlays; wire **`style-switcher.js`** and footer container if you want theme UI.
2. Add **`nebula-canvas`** + **`background.js`** if you want the same ambient background.
3. Reuse **`confirm-modal.js`** for any gated flow.
4. For multi-page apps, reuse **`swipe-nav.js`** pattern (adjust `BASE_PAGES` and completion hook).
5. For assessments, reuse **`EngineUIController`** + a single progress/results storage convention.
6. If using **Capacitor + unbundled ESM**, mirror **`copy-to-www.js`** vendor strategy or introduce a bundler.
7. Document your own **suite rules** and **IAP** (if any) beside this file.
